# План аналитики

Цель аналитики: понимать, как люди используют калькулятор, где возникают ошибки и какие функции нужно улучшать.

## Что собирать

1. Посещение сайта.
2. Устройство: телефон, планшет, компьютер.
3. Размер экрана.
4. Используемый раздел: расчет или сравнение.
5. Тип расчета: кредит или кредитная карта.
6. Нажатие кнопки расчета.
7. Нажатие кнопки PDF.
8. Ошибки ввода.
9. Ошибки JavaScript.
10. Переходы по рекламным или партнерским ссылкам.

## Что не собирать без необходимости

1. ФИО пользователя.
2. Телефон.
3. Email.
4. Полные данные конкретного кредита, если нет понятного согласия.
5. Персональные или банковские данные.

## Рекомендуемый стек

1. Яндекс Метрика для базовой веб-аналитики.
2. Sentry для технических ошибок JavaScript.
3. Простые события в коде: calculate, compare, pdf, validation_error.

## Принцип

Сначала собирать обезличенные события. Персональные данные не нужны для MVP.
